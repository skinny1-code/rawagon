How ZK Works
1. On your device: AuraMe generates master key from biometrics
2. Commitment: HMAC-SHA256(key, credentials) → stored on-chain
3. Proof: Generated on-device, proves attribute without revealing data
4. Merchant: Calls verifyProof() → gets true/false only
5. Anti-replay: Each proof hash marked spent — no replay attacks
